Point Of Failure Map

Where the system is most likely to break first, with immediate containment actions.

External API Dependency (OANDA)

High

Network/API errors can halt ingestion and distort freshness assumptions. Add retry with backoff, endpoint health checks, and stale-data alarms.

Database Write Contention

High

Concurrent inserts and duplicate handling can produce latency spikes. Add idempotent merge strategy and transaction batching safeguards.

Schema Drift Between Scripts

Medium

Legacy and active script overlap can create column mismatch failures. Enforce one canonical schema contract and preflight schema validation.

Model Drift / Regime Shift

High

Market condition changes invalidate static thresholds. Track rolling performance windows and automate retrain checkpoints with guardrails.

Frontend Callback Complexity

Medium

Large callback graph in app.py can cause regression risk. Split into modules and add callback-level tests for filters, charts, and exports.

Operational Observability Gaps

Low

Unstructured logs slow incident triage. Standardize logging schema and severity conventions across pipeline, trading, and retrain jobs.

Scalability Assessment

Current constraints and the best next scaling moves based on your folder structure and workflow patterns.

Codebase Structure Scaling

Promote archived production-worthy scripts into a consolidated src/ package with clear modules: ingest, features, signals, execution, and audit.

Data Throughput Scaling

Move from row fallback inserts to merge-staging patterns and chunked writes with metrics for write latency, duplicate rate, and failure ratio.

Model Lifecycle Scaling

Standardize model registry naming, add model metadata files, and track train dataset hash + feature set versions per model artifact.

UI Scaling

Break frontend/app.py into dashboards, callbacks, and chart-factory modules. This reduces coupling and speeds testability for future features.
