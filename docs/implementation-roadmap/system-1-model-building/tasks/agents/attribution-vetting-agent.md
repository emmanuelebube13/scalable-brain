## Verification Gates (Self-Check Before Handoff)

### MODEL-004 Gates
- [ ] Per-regime trade counts sum to aggregate trade count (within ±1 for rounding).
- [ ] Weighted per-regime metrics reconcile with global metrics within tolerance.
- [ ] No future regime used at trade entry: `regime_bar_time <= trade_entry_time` for all trades.
- [ ] `low_confidence` flag true for all cells with `trade_count < N_min` (configurable, default 20).
- [ ] Cells with zero trades in a regime are absent from the table (not zero-padded misleadingly).
- [ ] Trades with missing regime label are flagged/handled, not silently dropped.
- [ ] Bayesian shrinkage applied: shrunk cell metric is between cell raw metric and global metric.

### MODEL-005 Gates
- [ ] All 6 gates reject exactly at boundary and accept just inside:
  - PF 1.49 → rejected, PF 1.50 → accepted
  - Sharpe 0.79 → rejected, Sharpe 0.80 → accepted
  - MaxDD 25.1% → rejected, MaxDD 25.0% → accepted
  - WinRate 39.9% → rejected, WinRate 40.0% → accepted
  - Recovery 2.99 → rejected, Recovery 3.00 → accepted
  - OOS 59 months → rejected, OOS 60 months → accepted
- [ ] Cells with `low_confidence=true` always rejected regardless of metric values.
- [ ] JSON schema validation passes for both `regime_strategy_map.json` and `strategy_weights.json`.
- [ ] Weights sum to 1.0 per regime.
- [ ] Ranks are dense (no gaps: 1,2,3..., not 1,3,5).
- [ ] Ranking consistent with documented formula (verify top-ranked has highest composite score).
- [ ] Empty regimes explicitly listed in `empty_regimes` with warning — never silently omitted.
- [ ] Aggregate vetting still runs and produces the same outputs (backward compatible).
- [ ] OOS ≥ 60 months correctly measured across walk-forward folds (union of OOS fold spans).

## Failure Modes & Escalation

| Failure | Detection | Action | Escalate To |
|---------|-----------|--------|-------------|
| Regime label missing for many trade entries | >10% of trades have no regime match | Investigate `Fact_Market_Regime_V2` coverage, flag gap, possibly re-run MODEL-003 for missing range | `ml-regime-agent` (rework) |
| All strategies rejected in a regime | `empty_regimes` list non-empty | Emit explicit warning, surface rejection reasons | `serializer-infra-agent` (may need to relax via config tuning) |
| Low-confidence cells dominating | >50% of cells flagged low-confidence | Need more backtest data (OOS period too short) or lower N_min | `auditor-traceback-agent` (decide threshold adjustment) |
| Artifact schema violation | JSON schema validation fails | Block output, fix emitter code | Self (fix code) |
| Weights don't sum to 1 | Floating-point check | Fix normalization code | Self (fix code) |
| OOS measurement incorrect | OOS span < 60mo even when data goes back to 2005 | Verify walk-forward fold logic | Self (fix code) |

## Notes

- **Run in log-only mode first** for MODEL-005. Compute pass/fail and would-be artifacts without changing actual promotion. Only enable live gating after reviewing rejection rates and calibrating against historical data.
- The two JSON artifacts (`regime_strategy_map.json`, `strategy_weights.json`) are a **hard interface to Computer 2**. Any schema change is breaking — version `schema_version` and coordinate with the `serializer-infra-agent`.
- Small-sample bias is the dominant risk for MODEL-004. Bayesian shrinkage + `low_confidence` flagging are mandatory. MODEL-005 must never promote on a thin cell alone.
- The stricter gate may reject most or all current strategies. That's a feature, not a bug. Expose rejection reasons so the gate can be tuned with an audit trail.
- Granularity: H1, H4, D1 all supported. D1 is primary for MODEL-003; H1/H4 preserved for Layer 2/3 contracts.
