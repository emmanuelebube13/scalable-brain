# 🧠 Scalable Brain: Core System Architecture

## 📌 The Core Philosophy
The Scalable Brain is an **Instrument-Agnostic, Meta-Labeling Quantitative Pipeline**. 
Unlike retail trading bots that attempt to use Machine Learning to predict raw price movement, this system uses deterministic mathematical strategies to find potential edges, and uses Machine Learning strictly as a contextual risk-manager (Meta-Labeling).

Every asset (e.g., EUR_USD, GBP_USD) is processed independently through the first 4 layers before being aggregated at the Portfolio Level.

---

## 🏗️ The 6-Layer Execution Pipeline

### **Layer 0: Strategy Qualification Engine (Offline Sandbox)**
* **The Objective:** To mathematically prove that a trading algorithm possesses a structural edge before it is allowed to touch live market data.
* **The Process:** Strategies are hardcoded into Python and backtested against decades of historical OHLCV data. They are forced to use a standardized dynamic risk profile (e.g., 1.5 ATR Stop Loss).
* **The Exit Criteria:** To pass Layer 0, a strategy must generate a minimum number of trades, yield a mathematically positive **Expectancy**, and maintain a **Profit Factor > 1.15**. 
* **Transition:** Strategies that pass are marked as `PROMOTED ✅` and physically inserted into the `Dim_Strategy_Registry` SQL table. They now move to Layer 2.

### **Layer 1: Market Regime Detection (The Weather Report)**
* **The Objective:** To dynamically classify the current state of the market for each individual asset.
* **The Process:** A live Python script constantly analyzes recent volatility (ATR), trend strength (ADX), and session volume. It clusters the data to output a strict tag for the current timeframe.
* **Outputs:** Tags such as `Trending_HighVol`, `Sideways_LowVol`, or `MeanReverting`.
* **Transition:** This regime tag is pushed to the SQL database (`Fact_Market_Regime`) and broadcasted to Layer 3, acting as the context for the AI.

### **Layer 2: Live Strategy Bank (The Signal Generators)**
* **The Objective:** To constantly monitor live price action and generate theoretical trade ideas based on strict mathematical rules.
* **The Process:** The `PROMOTED` strategies from Layer 0 (e.g., Bollinger Band Mean Reversion) run continuously on incoming data. 
* **Outputs:** When a strategy's technical conditions are met, it outputs a raw, unverified `BUY` or `SELL` signal. 
* **Transition:** These signals are *not* sent to the broker. They are forwarded directly to the AI Gatekeeper in Layer 3.

### **Layer 3: The ML Meta-Labeler (The AI Gatekeeper)**
* **The Objective:** To contextualize and filter the raw signals generated by Layer 2 using Machine Learning (XGBoost/Random Forest).
* **The Process:** The ML model does not look at the price; it looks at the *Signal* (from Layer 2) and the *Regime* (from Layer 1). It calculates the historical probability of that specific strategy succeeding in that specific market weather.
* **Outputs:** A confidence score (0.0 to 1.0). 
* **Transition:** If the AI scores the trade > 0.75 confidence, the trade is `APPROVED` and passed to Layer 4. If it is < 0.75, the trade is `VETOED` and dies immediately.

### **Layer 4 & 4.5: Dynamic Risk & Portfolio Correlation (The Shield)**
* **Layer 4 (Trade Risk):** Before execution, the system dynamically calculates the exact Stop Loss and Take Profit distances using real-time Average True Range (ATR). Fixed pip/percentage stops are prohibited.
* **Layer 4.5 (Portfolio Exposure):** The system checks a rolling 30-day Correlation Matrix. 
* **Transition:** If an `APPROVED` EUR_USD Buy signal reaches this layer, but the portfolio is already holding a highly correlated GBP_USD Buy position, Layer 4.5 will block the trade to prevent hidden over-leveraging against the US Dollar. If all risk checks pass, the order is routed to the Oanda API for execution.

### **Layer 5: Telemetry & Execution (The Dashboard)**
* **The Objective:** To provide human oversight, system auditing, and visual performance tracking.
* **The Process:** A Python Streamlit (or Power BI) dashboard querying the SQL database in real-time.
* **Outputs:** Visualizes active regimes, maps out where the AI vetoed human-looking trades, tracks active drawdowns, and monitors live Expectancy. If a promoted strategy begins to decay in live markets, Layer 5 flags it for demotion back to Layer 0.
